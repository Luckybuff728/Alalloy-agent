# 非标准任务输入问题修复报告

**日期**：2026-03-08  
**版本**：v0.1.0 → v0.1.2  
**测试场景**：非常规输入下的任务确认机制与计算准确性

---

## 一、问题背景

测试任务为"计算凝固区间"和"计算平衡相组成"，验证在非常规输入下系统的确认机制与执行准确性，暴露以下三类问题：

| # | 问题描述 | 严重程度 |
|---|---|---|
| P1 | 成分含 TDB 不支持的元素时，系统静默剔除后继续计算，结果产生误导 | 高 |
| P2 | 用户未提供具体成分数值时，系统自行假设默认成分（如 Al-7Si-0.3Mg）并计算 | 高 |
| P3 | 成分单位未明确时，系统默认按 wt% 处理但不提示用户；微量元素原子分数精度截断导致副标题显示偏差 | 中 |

---

## 二、问题修复

### P1 — 含不支持元素时静默剔除 → 分元素性质差异化处理

**根本原因**：提示词要求"剔除不支持元素后继续计算"，AI 静默修改成分，结果并非用户原始合金的热力学数据。此外，过度严格的"完全中止"策略会阻断 `dataExpert` 推荐含 Sr/Ti 配方后用户进行热力学验证的正常工作流。

**修复**：新增**成分前置校验 C**，按不支持元素性质分两条路径处理：
- **路径 A（微量功能性添加元素：Sr ≤ 0.03%、Ti ≤ 0.2%、B ≤ 0.05%）**：根据上下文意图自动判断——若已明确是热力学分析（挂件触发、用户明确说热力学/凝固/相平衡、或 dataExpert 推荐后选分析），则**自动**剔除微量元素并重归一化后继续计算，仅输出说明文字告知用户；若意图不明确才展示选项询问。执行后结果中注明"不含 Sr/Ti，相平衡趋势可供参考"。
- **路径 B（主体合金元素：Cu > 0.5%、Zn > 0.5%、Ni、Cr 等）**：无论意图如何均完全中止，仅提供 `performance_prediction` + `generate_report`。

| 场景 | 修复前 | 修复后 |
|---|---|---|
| 含 Cu（主体元素）时 | 静默剔除 Cu，用 Al-1Mg 子系统计算 | 路径 B：中止，说明 Cu 不在 Al TDB，提示 ONNX 替代 |
| 含 Sr/Ti（微量功能元素），意图明确为热力学 | 静默剔除或完全中止，需额外点击 | 路径 A：自动剔除重归一化，输出说明后直接继续计算 |
| dataExpert 推荐含 Sr/Ti 配方后选热力学分析 | 完全中止，无法验证 | 路径 A：自动处理，无需额外操作，结果含免责说明 |
| 含 Sr/Ti，用户意图不明确 | 静默处理 | 路径 A：展示选项让用户确认 |

---

### P2 — 成分未明确时假设默认值 → 展示成分输入表单

**根本原因**：`analysisExpert` 在缺少成分时倾向于自行假设典型值后继续执行。

**修复**：新增**校验 A（成分是否含有具体数值）**——成分模糊时禁止继续，以 `form` 模式调用 `show_guidance_widget` 展示成分输入表单。CalphaMesh 任务展示 4 字段表单（Si/Mg/Fe/Mn，Al 余量）；`thinker.md` 补注无成分时仍路由到 `analysisExpert` 由其收集。

| 行为 | 修复前 | 修复后 |
|---|---|---|
| 无具体成分时 | 假设 Al-7Si-0.3Mg，直接计算 | 弹出成分表单（Si/Mg/Fe/Mn），等待用户填写 |

---

### P3 — 成分单位不提示 + 副标题显示偏差

**根本原因（a）**：提示词未明确单位处理规则，AI 对裸数字成分无清晰指引。  
**根本原因（b）**：`ToolStatusWidget.vue` 的 `_fmtComp` 将原子分数直接 ×100 展示为摩尔%，而非 wt%；微量元素原子分数截断为 4 位小数误差较大。

**修复**：新增**校验 B（单位是否明确）**，`Al-xSi-yMg` 标准符号默认 wt% 并告知用户；仅裸数字无单位时文字询问，不得继续计算。归一化步骤 4 要求保留 6 位小数。`_fmtComp` 重写为：检测到原子分数（maxVal ≤ 1.0）时，先 `wt_i = x_i × M_i` 换算再归一化，展示正确 wt% 副标题。

| | 修复前 | 修复后 |
|---|---|---|
| Si（7 wt%） | 显示 6.7（摩尔%×100） | 显示 7.0（wt%） |
| Mn（1 wt%） | 显示 0.05（摩尔%×100） | 显示 1.0（wt%） |

## 三、功能优化

### O1 — HITL 参数确认扩展至全部计算任务

**原因**：原系统仅 Scheil 触发 HITL，单点/温度扫描/二元相图/三元相图/ONNX 直接提交，参数错误只能等任务完成后重提。

**变更**：以下 6 项任务均在提交前弹出参数确认表单：

| 任务 | 主要可确认参数 |
|---|---|
| 单点平衡 | 温度（K）、成分原子分数 |
| 温度扫描 | 起止温度（K）、步数 |
| Scheil 凝固 | 起始温度（K）、步长（K） |
| 二元相图 | 组元对、温度范围 |
| 三元截面 | 三元组元、等温温度（K） |
| ONNX 性能预测 | 模型 UUID、全部 15 种元素 wt% |

## 四、报告生成质量修复

### R1 — 报告只包含已计算内容

**根本原因**："固定章节不得增删"+ 数据缺失规则强制写占位文字，导致未执行 CalphaMesh 时报告仍包含热力学章节。

**修复**：改为条件渲染——各子节只在对话历史中存在对应工具实际返回结果时才写入，否则**完全省略**（不写"待补充"等任何占位文字）；删除 Cu/Zn 趋势分析规则；意图说明改为动态。

---

### R2 — CalphaMesh 计算结果含分析结论

**根本原因**：热力学特性各子节仅列原始数据点，无材料学分析结论。

**修复**：每个计算类型新增分析结论（1–2 句）要求：

| 计算类型 | 分析结论要求 |
|---|---|
| Scheil 凝固 | 凝固区间工程评价（< 50 K 优/50–70 K 良/> 70 K 热裂高） |
| 单点平衡 | 各相对力学性能的影响，该温度是否适合热处理/服役 |
| 温度扫描 | 固溶处理温度窗口、时效温度参考 |
| 二元/三元相图 | 候选成分的相图位置（亚共晶/共晶/过共晶），是否需要调整 |
| 热力学性质 | 热容峰与相变对应关系，对工艺热管理的意义 |

## 五、ONNX 成分表单与校验逻辑修复

### F1 — 欢迎页示例问题更新

更新 4 条示例问题为实际测试场景，补回 ONNX 性能预测入口。

---

### F2 — ONNX 性能预测成分表单确认

**问题**：无成分时输出纯文字要求手填 15 个元素，体验差。

**修复**：校验 A 新增 ONNX 专用 8 字段表单，按意图关键词自动选择：

| 意图 | 触发关键词 | 表单字段 |
|---|---|---|
| 🔵 性能预测 | 预测/力学性能/强度/延伸率/ONNX | Si / Cu / Mg / Fe / Mn / Zn / Ti / Sr（Al 余量，其余 6 元素自动设 0） |
| 🟠 热力学分析 | 热力学/相图/凝固/Scheil/相平衡 | Si / Mg / Fe / Mn（Al 余量） |

---

### F3 — Check C 误判 ONNX 成分（关键漏洞）

**问题**：原规则"执行任何工具前须通过三项校验"，导致 ONNX 预测含 Cu 的 A380 时，Check C 误报"Cu 不在 TDB"并中止，而 ONNX 根本不依赖 TDB。

**修复**：前置规则明确区分——ONNX 只需校验 A + B；CalphaMesh 需 A + B + C；Check C 标题加注"仅 CalphaMesh 任务执行，ONNX 直接跳过"。
