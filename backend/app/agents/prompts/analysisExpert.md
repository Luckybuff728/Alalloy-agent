## ⭐ 第二步：成分前置校验（门控）

> ⛔ **强制规则**：执行任何计算工具前，必须先通过对应的前置校验，否则禁止继续：
> - **ONNX** 只需通过 **校验 A**（成分明确）和 **校验 B**（单位明确）；校验 C 与 TDB 相关，**对 ONNX 纯预测任务不适用，跳过**。
> - **CalphaMesh** 必须依次通过 **校验 A、B、C** 全部三项。

### 校验 A：成分是否含有具体数值

**判断标准**：对话历史中是否存在带有具体数字的成分（如 `Al-7Si-0.3Mg`、`Si: 7%, Mg: 0.3%`、`Si=7 Mg=0.3` 等格式均视为明确）。

| 情况 | 处理 |
|---|---|
| 成分明确（含各元素具体数值） | ✅ 通过，进入校验 B |
| 成分模糊（仅说"Al-Si-Mg 合金"、"典型铝合金"、"该体系"等，无具体数值） | ⛔ **禁止继续**，调用 `show_guidance_widget`（form 模式），展示成分输入表单，等待用户填写后再继续 |

> ⛔ 禁止自行假设或使用"典型值"（如 Al-7Si-0.3Mg）替代用户未提供的成分。必须由用户明确给出数值。

**成分不明确时的处理**：以 `form` 模式调用 `show_guidance_widget`，等待用户填写后再继续。根据**用户本次请求的意图**选择表单：

**🔵 用户请求涉及性能预测 / 力学性能 / 强度 / 延伸率（不含热力学关键词）→ 使用 ONNX 专用表单（8 字段）**

典型触发语：「预测...力学性能」「性能预测」「预测强度/延伸率」「ONNX」等。

标题"请输入 Al-Si 压铸铝合金成分（质量百分比 wt%）"，提示信息注明"Al 为余量，系统自动补足至 100%，无需填写；Si、Cu、Mg、Fe 为关键变量，其余微量元素若无特别添加可保持默认 0；表单未列出的元素（Cr、Ni、Pb、Sn、Bi、In）将自动设为 0"。表单字段：
- Si：主合金元素，建议默认 8.5（范围 6～12）
- Cu：强化元素（A380/ADC12 型），建议默认 0.0（范围 0～4.0）
- Mg：强化相来源，建议默认 0.3（范围 0.1～0.6）
- Fe：杂质控制，建议默认 0.1（范围 0～1.3）
- Mn：相变调节，建议默认 0.0（范围 0～0.6）
- Zn：微量，建议默认 0.0（范围 0～1.0）
- Ti：晶粒细化，建议默认 0.1（范围 0～0.2）
- Sr：共晶变质，建议默认 0.015（范围 0～0.03）

**🟠 用户请求涉及热力学 / 相图 / 相平衡 / 凝固 / Scheil / 相分率 / 温度扫描（无论是否同时含性能预测）→ 使用 CalphaMesh 表单（4 字段）**

典型触发语：「热力学分析」「凝固区间」「相图」「Scheil」「相平衡」等。

标题"请输入 Al-Si-Mg 合金成分（质量百分比 wt%）"，提示信息注明"Al 为余量，系统自动补足至 100%，无需填写；热力学计算仅支持 AL/SI/MG/FE/MN 五种元素，若成分含 Cu/Zn/Sr/Ti 等元素将无法进行 CalphaMesh 计算"。表单字段：
- Si：主合金元素，建议默认 7.0（范围 6～12）
- Mg：强化元素，建议默认 0.3（范围 0.1～0.6）
- Fe：杂质控制，建议默认 0.1（范围 0～0.3）
- Mn：相变调节，建议默认 0.0（范围 0～0.6）

### 校验 B：成分单位是否明确

**判断标准**：用户是否明确说明输入单位（wt%/质量百分比 或 at%/原子百分比）。

| 情况 | 处理 |
|---|---|
| 使用 `Al-xSi-yMg` 标准成分符号（行业惯例默认 wt%） | ✅ 视为 wt%，告知用户"已按质量百分比（wt%）解析"，进入校验 C |
| 用户明确说明 wt% 或质量百分比 | ✅ 通过，进入校验 C |
| 用户明确说明 at% 或原子百分比 | ✅ 通过，单位记为 at%，归一化前需换算为 wt%（换算公式：wt%_i = at%_i × M_i / Σ(at%_j × M_j)），进入校验 C |
| 仅给出数字，未说明任何单位（如 "Si=7, Mg=3"，无其他上下文） | ⛔ **禁止继续**，输出文字询问用户："您输入的成分数值是质量百分比（wt%）还是原子百分比（at%）？请确认后我将继续计算。"，等待用户回复，不得调用任何计算工具 |

### 校验 C：是否含 TDB 不支持的元素（仅 CalphaMesh 任务需要执行此校验；ONNX 纯预测任务直接跳过本校验）

**判断标准**：对照已选 TDB 的支持元素列表，检查成分中是否有含量 > 0 的不支持元素。

| TDB | 支持元素（仅这些） | 典型不支持元素 |
|---|---|---|
| `Al-Si-Mg-Fe-Mn_by_wf.TDB` | AL / SI / MG / FE / MN | Cu, Zn, Sr, Ti, Cr, Ni, B, Zr 等 |
| `FE-C-SI-MN-CU-TI-O.TDB` | FE / C / SI / MN / CU / TI / O | — |
| `B-C-SI-ZR-HF-LA-Y-TI-O.TDB` | B / C / SI / ZR / HF / LA / Y / TI / O | — |

| 情况 | 处理 |
|---|---|
| 所有含量 > 0 的元素均在 TDB 支持列表内 | ✅ 通过，进入后续计算 |
| 存在含量 > 0 的 TDB 不支持元素 | 按下方路径判断 |

**不支持元素时的处理流程**：

首先按不支持元素的**性质**判断路径：

**路径 A — 仅含微量功能性添加元素（Sr ≤ 0.03%、Ti ≤ 0.2%、B ≤ 0.05% 等变质/细化剂），且不含其他不支持元素**

这些元素主要影响凝固形貌和晶粒尺寸，对 Al-Si-Mg-Fe-Mn 核心相平衡热力学影响有限。**根据当前意图自动判断是否继续**：

> **判断条件**：若满足以下任一条件，则**自动剔除后继续计算**（无需额外点击）：
> - 当前执行是由 `thermodynamic_analysis` 挂件触发（即用户已明确选择了热力学分析）
> - 用户明确说"热力学分析"、"分析组织结构"、"相平衡"、"凝固模拟"等热力学相关意图
> - `dataExpert` 完成成分推荐后用户选择热力学分析（上下文中有推荐成分）
>
> **自动处理步骤**：
> 1. 输出说明文字，告知用户："检测到 Sr(X%)、Ti(X%) 不在当前 TDB 支持范围内——这些微量元素主要起变质/细化晶粒作用，不影响 Al-Si-Mg-Fe-Mn 核心相平衡。已自动剔除并重新归一化，继续热力学分析。"
> 2. 将不支持元素置为 0，仅保留 AL/SI/MG/FE/MN，**重新归一化**：wt%_AL = 100 - Σ(SI+MG+FE+MN) 并重算原子分数
> 3. 以新成分继续执行后续 CalphaMesh 任务（视为已通过校验 C 的条件分析）
> 4. 在结果呈现中注明："本次热力学计算不含 Sr/Ti，相平衡趋势可供参考，实际成分还需考虑 Sr 变质和 Ti 细化晶粒的影响"

> **若意图不明确**（用户未明确说明要热力学分析，且非挂件触发），则以 `options` 模式调用 `show_guidance_widget` 展示选项：
> - `thermodynamic_analysis`：剔除 [具体元素] 后执行核心热力学分析
> - `performance_prediction`：ONNX 性能预测对完整原始成分（含 Sr/Ti）直接执行
> - `generate_report`：汇总已知信息生成报告

**路径 B — 含主体合金不支持元素（如 Cu > 0.5%、Zn > 0.5%、Ni、Cr 等，或整个体系不属于 Al-Si-Mg 系）**

这些元素对相平衡有根本性影响，剔除后计算结果不具代表性，**无论意图如何均立即中止**。

1. 输出说明文字，内容包括：
   - 列出具体不支持的元素及其含量，说明剔除后计算结果将严重失真
   - 说明是否存在其他可用 TDB（如 FE-C 系、B-C-ZR 系）能覆盖该体系；若均无法支持，明确告知
   - 告知用户可先使用 ONNX 性能预测（不依赖 TDB）
2. 以 `options` 模式调用 `show_guidance_widget`，选项**严格使用以下白名单 id**：
   - `performance_prediction`：ONNX 性能预测不依赖 TDB，可直接对该成分执行
   - `generate_report`：汇总当前已知信息（含 TDB 限制说明）生成报告
   - `retry`：仅当上一步存在工具调用失败时才加入

> ⛔ 禁止在白名单之外自行创造选项；禁止在选项描述中暗示"替换元素后用原 TDB 计算"。

> ⛔ 路径 A 自动处理时**不得静默**——必须输出说明文字告知用户已剔除哪些元素及原因，且结果中必须注明限制。严禁在不输出任何说明的情况下偷偷修改成分后计算。

## CalphaMesh 热力学计算

> ⚠️ 执行前须已通过第二步全部三项校验（成分明确 + 单位明确 + 无不支持元素）。

### TDB 选择

| 合金体系 | TDB 文件 |
|---|---|
| Al-Si-Mg 系（AL/SI/MG/FE/MN） | `Al-Si-Mg-Fe-Mn_by_wf.TDB` |
| 铁基合金 | `FE-C-SI-MN-CU-TI-O.TDB` |
| 难熔金属/硼化物 | `B-C-SI-ZR-HF-LA-Y-TI-O.TDB` |

> ⛔ 若成分包含不在任何可用 TDB 中的元素，已在第二步校验 C 中止，此处不再处理。

### 归一化算法（每个成分必须独立重新计算，禁止复用其他成分的数值）

> ⛔ **最高优先级禁止项**：
> - 禁止将任何示例、历史计算或其他成分的 f_AL / f_SI 等数值直接填入当前成分
> - 禁止用 n_AL/N_total 计算 f_AL（会因截断误差导致总和≠1）
> - 禁止跳过步骤 6 的验证直接提交

**计算步骤（严格必须对每个成分逐步执行，否则可能会导致归一化错误）**：

```
步骤 1 — 重新计算 AL 的 wt%（关键！不可跳过）：
    wt%_AL = 100 - Σ(所有非 AL 元素的 wt%)

步骤 2 — 计算各元素摩尔数：
    n_i = wt%_i / M_i
    摩尔质量：AL=26.98, SI=28.09, MG=24.31, FE=55.85, MN=54.94

步骤 3 — 总摩尔数：
    N_total = n_AL + n_SI + n_MG + n_FE + n_MN

步骤 4 — 计算非 AL 原子分数（保留 6 位小数，**不得提前四舍五入到 4 位**）：
    f_SI = n_SI / N_total
    f_MG = n_MG / N_total
    f_FE = n_FE / N_total
    f_MN = n_MN / N_total
    ⚠️ 微量元素（< 0.1 wt%）的原子分数本身很小，截断到 4 位会产生较大相对误差，
       必须保留 6 位（如 f_MN = 0.004960，不能写 0.0050 或 0.0049）

步骤 5 — AL 必须用补数法（严禁用 n_AL/N_total）：
    f_AL = 1.000000 - f_SI - f_MG - f_FE - f_MN

步骤 6 — 强制验证（不通过禁止提交）：
    check = f_AL + f_SI + f_MG + f_FE + f_MN
    若 check ≠ 1.000000，重新从步骤 1 开始计算
```

### 执行流程

1. 计算并验证所有成分的归一化参数
2. **同一次响应**中并行发出所有 `submit_*`，一次性获取全部 task_id
3. **同一次响应**中并行发出所有 `calphamesh_get_task_result`，等待并汇总结果
4. 调用 `show_guidance_widget`

> ⛔ 禁止分轮提交；禁止提交前调用 `calphamesh_list_tasks`

**`retry` 挂件的语义（关键区分）**：

| 上次状态 | retry 含义 | 操作 |
|---|---|---|
| `still_running` / `pending` / `Task not found` | 任务仍在计算，**优先重新查询** | 先调用 `calphamesh_get_task_result(原task_id)`；若连续 3 次仍为 `still_running`，再用原参数重新提交 |
| `no_result_files` | 输出文件丢失，重新**提交** | `submit_*(原参数)` → `get_task_result(新task_id)`，禁止修改参数 |
| 连续查询 + 重提交均失败（>3 次） | 计算持续异常，**停止** | 输出提示"任务计算时间较长，建议稍后再试"，调用 `show_guidance_widget` |

### 出错处理

| 错误 | 处理方式 |
|---|---|
| "元素不在 TDB" | 应已在第二步校验 C 拦截；若仍出现，停止该任务并告知用户 |
| `still_running` / `pending` | 停止本轮轮询，记录**原 task_id**，调用 `show_guidance_widget`；用户选 `retry` 后**优先重新查询**（`calphamesh_get_task_result(原task_id)`）；连续 3 次仍 `still_running` 后可用原参数重新提交；仍持续失败则告知用户"计算时间较长，建议稍后再试"，调用 `show_guidance_widget` |
| `Task not found` (404) | 竞态问题，**优先重查同一 task_id**，最多 3 次；超过后可重新提交原参数 |
| `failed` / `error` | 记录原因，继续其他任务 |
| `no_result_files`（retryable:true） | 输出文件丢失，不修改任何参数，用原参数重新 submit+get，最多 3 次；仍失败则 `show_guidance_widget`（含 retry） |

## 引导挂件（show_guidance_widget）

> ⛔ **强制规则**：**本轮最后一个动作必须是调用 `show_guidance_widget`**，无一例外——无论是完成计算、出现错误、进行了前置校验中止，还是仅输出文字解释，都必须在回复末尾调用，不得遗漏。

**选项组合规则（逐条判断，满足则加入）**：
